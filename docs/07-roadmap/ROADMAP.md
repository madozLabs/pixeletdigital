# Roadmap — Pixel&Digital Platform

## Principes

La roadmap sépare clairement l'expérience publique, le Workspace interne et les modules métier avancés. Chaque phase doit produire une valeur utilisable et observable. Aucun module ne sera développé uniquement parce qu'il paraît impressionnant.

## Phase 0 — Fondations produit

Statut : en cours.

Livrables :

- gouvernance CTO ;
- Project Bible ;
- architecture de marque ;
- périmètre MVP ;
- rôles et permissions ;
- parcours publics et internes ;
- roadmap ;
- décisions ouvertes documentées.

Critère de sortie : vision cohérente, périmètre compréhensible et arbitrages critiques identifiés.

## Phase 1 — Direction de marque et UX

Livrables :

- Brand Bible Pixel&Digital ;
- Brand Bible Kwaliti Print ;
- fondations et teasers Studio et Formation, sans lancement complet au MVP sauf changement de périmètre approuvé ;
- architecture de l'information ;
- wireframes des parcours prioritaires ;
- règles d'immersion, mouvement, 3D et accessibilité ;
- Design System conceptuel ;
- stratégie éditoriale et de preuve.

Critère de sortie : les univers sont différenciés sans perdre la cohérence de l'écosystème.

## Phase 2 — Architecture technique

Démarre uniquement après validation explicite du propriétaire.

Livrables :

- exigences fonctionnelles et non fonctionnelles consolidées ;
- choix de stack documentés ;
- modèle de données ;
- architecture applicative ;
- stratégie d'authentification et d'autorisation ;
- stockage médias ;
- SEO, performance, observabilité et sécurité ;
- environnements et CI/CD ;
- ADR pour les décisions majeures.

Critère de sortie : une architecture minimale, testable et capable d'évoluer sans sur-ingénierie.

## Phase 3 — MVP public Pixel&Digital

Objectifs : établir une présence remarquable et convertir les visiteurs.

Périmètre :

- accueil ;
- agence ;
- univers et expertises ;
- services prioritaires ;
- études de cas et réalisations ;
- équipe ;
- ressources ou actualités ;
- contact, rendez-vous et demande de devis ;
- SEO, analytics, performance et accessibilité.

Critère de sortie : expérience publique stable, rapide, responsive et mesurable.

## Phase 4 — Kwaliti Print

Objectifs : lancer un univers distinct, immersif et commercialement exploitable.

Périmètre :

- identité et navigation propres ;
- catégories d'impression et personnalisation ;
- réalisations ;
- matières et finitions ;
- demande de devis enrichie ;
- médias immersifs et 3D progressive lorsque pertinente ;
- alternative légère sur mobile ou appareils limités.

Critère de sortie : Kwaliti Print peut convertir indépendamment tout en restant relié à l'écosystème.

## Phase 5 — Workspace MVP

Objectifs : permettre aux équipes de gérer le contenu et les demandes entrantes.

Périmètre :

- authentification ;
- utilisateurs, rôles et périmètres ;
- CMS structuré multi-univers ;
- bibliothèque médias ;
- cycle éditorial canonique Brouillon, En revue, Planifié si nécessaire, Publié et Archivé ;
- leads, qualification, affectation et prochaines actions ;
- activité récente et notifications essentielles ;
- audit des actions sensibles.

Critère de sortie : trois flux fiables — publication, traitement d'un lead et administration des accès.

## Phase 6 — Studio

Sous réserve du nom, de l'identité et des besoins opérationnels validés.

Périmètre potentiel :

- vitrine audiovisuelle ;
- showreels et projets ;
- demandes de brief ;
- réservation ou disponibilité du studio ;
- catalogue de formats ;
- gestion légère des productions.

## Phase 7 — Formation

Sous réserve du positionnement et du nom validés.

Périmètre potentiel :

- catalogue de formations ;
- fiches programmes ;
- sessions et préinscriptions ;
- formateurs ;
- suivi administratif ;
- paiements, espace apprenant et attestations dans une phase ultérieure.

## Phase 8 — Opérations agence

Démarre après observation des workflows réels.

Périmètre potentiel :

- opportunités et devis ;
- clients ;
- briefs ;
- projets ;
- tâches et responsabilités ;
- livrables et validations ;
- portail client ;
- rapports ;
- automatisations ciblées.

## Phase 9 — Intelligence et optimisation

Uniquement lorsque les données et processus sont suffisamment fiables.

Périmètre potentiel :

- suggestions de contenu ;
- recherche sémantique ;
- synthèse d'activité ;
- assistance aux briefs ;
- classification de leads ;
- prévision commerciale ;
- automatisations assistées par IA avec validation humaine.

## Exclusions persistantes

- messagerie interne complète ;
- visioconférence ;
- comptabilité générale ;
- réseau social d'entreprise ;
- moteur générique no-code ;
- microservices sans justification ;
- personnalisation algorithmique opaque au lancement.

## Gates de décision

Chaque phase exige :

1. périmètre explicite ;
2. critères d'acceptation ;
3. risques identifiés ;
4. validation UX et sécurité proportionnée ;
5. tests adaptés ;
6. revue avant fusion et déploiement.

## Décision CTO

La prochaine phase active est la direction de marque et l'UX. L'architecture et le développement restent bloqués jusqu'au feu vert explicite du propriétaire.
